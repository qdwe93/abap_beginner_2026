## 2. 검토 기준 (심각도 분류)

기존 검토 컨벤션(90~95 문서)과 동일하게 P0/P1/P2로 분류한다.

### P0 — 원칙 위반·오류 (반드시 수정)

1. **창작 코드**: 슬라이드의 코드 조각이 교재 코드의 부분(생략 표시 `...` 허용)이 아닌 경우. 문법 오류 코드 포함.
2. **퀴즈 정답 노출**: 퀴즈 슬라이드(Q01~Q19, R2/R3, 미니퀴즈①, 종합실습 TODO)에 부록 A 정답 코드·정답 텍스트가 보이는 경우.
3. **확정 설계 위반** (94 지시서 6장): classic 본문 + ✨신문법은 보조 박스 / `REUSE_ALV_GRID_DISPLAY` 유지 / 취소 기능 없음(TP-5 확장 1로만) / X 수정 차단은 리터럴 메시지·**메시지 023 미출현** / MAX+1 채번 / DEC 단가·금액 미저장·메인 COMMIT / POPUP_GET_VALUES / 잠금 없음 / LIST→U19→MAIN 복사 체인 / `##` 학생 번호 표기 / U17 5단계 TODO(주: 교재 기준은 TP TODO 구성) / U16 DEFAULT 3종.
4. **위험 패턴 3종이 "정상 예"로 등장**: WRITE 안 산술식 / CONCATENATE에 숫자형 직접 사용 / ORDER BY에 집계함수 직접 사용. (❌→✅ 대비 슬라이드의 ❌ 측 표시는 정상.)
5. **사실 오류**: 프로그램명·T-code·테이블/필드명·메시지 번호·시스템 필드 동작 설명이 교재와 다른 경우.

### P1 — 내용 불일치·누락

1. 유닛 헤더의 **시간이 커리큘럼·교재 헤더와 불일치** (총 25개 세그먼트).
2. **운영 장치 누락/오배치**: 타이머 배너 — Day 1 U07 25/43분, Day 3 U16 30/48분·U19 25분·TP-2 15/25/35분. (Day 2는 강제 체크포인트 없음이 정상.)
3. 퀴즈 슬라이드의 요구사항·제한시간·힌트가 교재 ✍️ 절과 다르거나, E/D 확장(퀴즈집 Uxx-E/D) 안내 누락.
4. 흔한 에러 슬라이드가 강사가이드 "자주 막힘"과 어긋남.
5. 발표자 노트 누락 또는 출처가 틀림.
6. 캡처목록 ↔ 슬라이드 📷 자리표시자 불일치 (ID, 슬라이드 번호, 캡션 내용).

## 3. 검토 방법 (권장 절차)

### 3-1. 텍스트 덤프 (본문+노트 대조용)

```powershell
$env:PYTHONUTF8 = "1"
markitdown "50_슬라이드\SLIDES_Day1.pptx" > day1_dump.md   # scratchpad에 저장
```

markitdown이 슬라이드 텍스트와 발표자 노트를 함께 덤프한다(`pip install "markitdown[pptx]"` 필요). 덤프를 교재 절 단위로 대조하는 것이 효율적이다. 코드 조각 대조는 덤프의 코드 줄을 교재에서 검색(부분 문자열 여부 확인)하는 방식.

### 3-2. 렌더 (육안 확인용)

이 환경엔 LibreOffice/pdftoppm이 없다. **PowerPoint COM export**가 동작 확인됨:

```powershell
$pp = New-Object -ComObject PowerPoint.Application
$pres = $pp.Presentations.Open($pptx, $true, $false, $false)
$pres.Export($outDir, "PNG", 1600, 900)
$pres.Close(); $pp.Quit()
```

export 파일명은 `슬라이드N.PNG`(한글). Pillow로 3×4 컨택트시트로 묶어 Read 도구로 검토하고, 의심 슬라이드만 원본 PNG로 확대 확인한다.

### 3-3. 진행 순서

1. Day 1부터 Day별로: 텍스트 덤프 ↔ 교재/퀴즈집/강사가이드 대조 → 렌더 육안 → 결함 기록.
2. 전 Day 공통 항목 일괄 점검: 시간표 25개, `##` 표기, 메시지 023 검색(0건이어야), 위험 패턴 3종 검색, 캡처목록 대조, Day 간 브릿지(예고↔실제).
3. **검토보고서 `99_슬라이드_검토보고서.md` 작성** — 결함 표: `ID / Day-슬라이드번호 / 심각도 / 증상 / 근거(교재·가이드 절) / 제안 수정`.
4. P0·P1은 수정 반영(아래 4절), P2는 목록으로 사용자 승인을 받은 뒤 적용.
5. 수정 후 재빌드·재검증·해당 슬라이드 재렌더 확인 → 커밋.

## 4. 수정 방법 (반드시 이 방식으로)

- **pptx를 직접 편집하지 말 것.** 수정은 `50_슬라이드/_src/day{1,2,3}_slides.js`에서 하고 재빌드한다 — 스크립트가 원본이다.
- 재빌드 전 각 스크립트 상단 `OUT` 경로 확인: **day1/day2는 `C:\workspace\...`로 되어 있어 이 환경(`C:\Workspaces\abap_beginner_2026`)과 다르다.** 재빌드가 필요하면 경로를 바로잡아 커밋한다(day3는 이미 올바름).

```powershell
npm install -g pptxgenjs            # 전역에 없으면 (세션마다 사라질 수 있음)
$env:NODE_PATH = (npm root -g)
node "50_슬라이드\_src\day3_slides.js"
```

- 재빌드 후 구조 검증(**PYTHONUTF8=1 필수** — 없으면 cp949 오탐):

```powershell
$env:PYTHONUTF8 = "1"
python <scripts>\office\validate.py "50_슬라이드\SLIDES_Day3.pptx"
```

  ⚠️ pptx 스킬의 scripts 폴더는 원 위치에서 직접 실행이 안 될 수 있다(경로 문제). 그 경우 **Git Bash의 `cp -r`로 스킬 `scripts` 폴더를 scratchpad에 복사**해 실행한다(PowerShell Copy-Item은 해당 경로를 못 읽었음). 의존성: `pip install defusedxml lxml "markitdown[pptx]" Pillow`.
- 수정한 슬라이드는 3-2 방식으로 재렌더해 육안 확인한다.
