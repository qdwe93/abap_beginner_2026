## 2. P0 — 반드시 수정할 항목

### 2-1. Day 1 Q02 WRITE 산술식

대상: `10_교재_Day1.md` U02 Q02 스켈레톤과 부록 A Q02 정답

현재 문제:

```abap
WRITE: / '금액    :', gv_amount,
       / '할인    :', gv_discount,
       / '결제금액:', gv_amount - gv_discount.
```

수정 원칙:

- `gv_payment TYPE p LENGTH 10 DECIMALS 2`를 선언한다.
- WRITE 전에 `gv_payment = gv_amount - gv_discount.`로 계산한다.
- WRITE에는 `gv_payment`만 전달한다.
- 학생이 직접 작성할 TODO 범위와 부록 A 정답이 서로 맞아야 한다.
- 기대 결과 1,260,000.00은 유지한다.

### 2-2. 퀴즈집 D10 WRITE 산술식

대상: `20_퀴즈집.md` D10 문제·정답·해설

현재 문제:

```abap
WRITE: / '삭제 건수:', gv_before - gv_after.
```

수정 원칙:

- `gv_deleted TYPE i`를 선언한다.
- `gv_deleted = gv_before - gv_after.`를 먼저 계산한다.
- WRITE에는 `gv_deleted`를 전달한다.
- D10이 `학생용_스켈레톤/U10_ZEDU##_U10.abap`에 이어서 작성하는 방식은 유지한다.

### 2-3. 학생용 U19를 실제 LIST 복사본 상태로 복원

대상: `40_패키지/학생용_스켈레톤/U19_ZEDU##_U19.abap`

현재 상태:

- 선택화면 4종, 최근 90일 INITIALIZATION, WHERE, `p_stat` 필터, 금액 계산, SORT는 존재한다.
- 그러나 `FORM show_alv`의 기본 `REUSE_ALV_GRID_DISPLAY` 호출 전체가 TODO다.
- `i_grid_title = '주문 목록'`도 실제 코드에 없다.

수정 목표:

- 다음 항목은 완성 상태로 제공한다.
  - `i_callback_program`
  - `i_structure_name`
  - `is_layout`
  - `i_grid_title = '주문 목록'`
  - `t_outtab`
  - 예외 매핑과 `sy-subrc` 처리
- 학생 TODO는 다음 콜백 학습 범위만 남긴다.
  - `i_callback_pf_status_set = 'SET_STATUS'`
  - `i_callback_user_command = 'USER_COMMAND'`
  - `FORM set_status`
  - `FORM user_command`
- 교재 U19의 “기존 ALV 호출에 콜백 2줄 추가” 흐름과 같아야 한다.

### 2-4. TP 학생용 스켈레톤과 복사 체인 정합화

대상:

- `40_패키지/학생용_스켈레톤/TP_ZEDU##_MAIN.abap`
- `40_패키지/README.md`

확정 복사 체인:

```text
ZEDU##_LIST → ZEDU##_U19 → ZEDU##_MAIN
```

수정 원칙:

- 이 복사 체인은 변경하지 않는다.
- TP 파일을 새 프로그램 생성용 독립 골격처럼 안내하지 않는다.
- 권장 방식:
  1. TP 스켈레톤을 U19→MAIN 복사 직후 상태와 맞춘다.
  2. 조회·ALV·콜백·PF-STATUS 참조는 유지하고 TP-2~4의 create/change/delete FORM만 단계별 TODO로 둔다.
  3. 파일 머리말과 README에 “U19를 MAIN으로 GUI Status·Text Elements와 함께 복사한 뒤 사용하는 복구/대조용 소스”라고 명시한다.
- PF-STATUS는 `.abap` 텍스트만으로 생성되지 않으므로, SE38 복사 시 GUI Status 체크가 필수라는 문구를 유지한다.

## 6. 절대 변경하지 말아야 할 확정 설계

1. 기본 과정에서 취소 기능을 구현하지 않는다.
2. X 수정 차단은 리터럴 메시지이며 메시지 023을 만들지 않는다.
3. 리터럴 메시지와 메시지 클래스를 의도적으로 혼용한다.
4. MAX+1, DEC 단가, 금액 미저장, 메인 COMMIT, POPUP_GET_VALUES, 잠금 없음은 교육용 확정 설계다.
5. `REUSE_ALV_GRID_DISPLAY`를 유지한다.
6. 본문은 classic 문법, 신문법은 보조·확장 영역이다.
7. LIST → U19 → MAIN 복사 체인을 유지한다.
8. Q03·Q05는 백지 코딩이다.
9. U08은 15분 운영 루틴으로 유지한다.
10. 스크린샷 자리표시자와 코드의 `##` 학생 번호 표기를 유지한다.
11. U17 학생용 5단계 TODO 구성을 유지한다.
12. U16 DEFAULT 3종과 `gv_amount`를 유지한다.
13. D12 별칭 정렬, D19·확장 2 문자 변환, E10·D10 U10 연계 방식을 유지한다.
14. 패키지 U02 강사용 완성본에는 부가세 출력 줄을 추가하지 않는다.
