### SAP GUI 기본 조작

- **트랜잭션 코드(T-code)**: 화면 좌상단 명령창에 입력하는 "화면 바로가기". 오늘 쓸 것: `SE38`(프로그램), `SE11`(딕셔너리), `SE16N`(데이터 조회)
- 명령창 접두어: `/n` + T-code = 현재 창에서 이동, `/o` + T-code = 새 창(세션)으로 열기 (최대 6개)
- 자주 쓰는 키: **F8 실행**, F3 뒤로, F1 도움말(커서 위치 기준), F4 입력 가능한 값 목록
- SAP은 ERP(전사적 자원관리) 시스템이고, ABAP은 그 위에서 돌아가는 프로그램을 만드는 언어입니다. FI(회계)·SD(영업)·MM(구매) 같은 모듈 이야기는 강의 중 쉬는 시간마다 조금씩 소개합니다.

## U01. SE38 첫 프로그램과 WRITE (09:40~10:30)

### 🎯 목표
- 리포트 프로그램을 생성·저장·활성화·실행할 수 있다
- ABAP 문장의 기본 규칙(마침표, 주석, 체인)을 안다

### 📖 개념

**리포트 프로그램** = 실행하면 (선택화면에서 조건을 받고) 로직을 수행해 결과 목록을 보여주는 가장 기본적인 ABAP 프로그램. 우리가 3일간 만드는 것은 전부 리포트 프로그램입니다.

**개발 사이클**: 작성 → **저장**(Ctrl+S) → **활성화**(Ctrl+F3, 문법검사+실행가능 상태로) → **실행**(F8). "저장했는데 왜 반영이 안 되죠?" → 활성화를 안 했기 때문. 3일 내내 쓸 리듬입니다.

ABAP 문장 규칙 3가지:
1. 모든 문장은 **마침표(.)** 로 끝난다
2. **대소문자를 구분하지 않는다** (단, 따옴표 안 문자열은 구분)
3. 주석: 줄 맨 앞 `*` = 그 줄 전체, 문장 중 `"` = 그 뒤부터 주석

### 💻 따라치기 — 프로그램 생성

1. 명령창에 `SE38` 입력 → Enter 📷
2. 프로그램 이름 `ZEDU##_U01` 입력 → [Create]
3. 속성 창: Title `첫 프로그램`, Type `Executable Program` → [Save]
4. 패키지 창 → **[Local Object]** 클릭
5. 에디터에 아래 입력:

```abap
*&---------------------------------------------------------------------*
*& Report ZEDU##_U01 - 나의 첫 ABAP 프로그램
*&---------------------------------------------------------------------*
REPORT zedu##_u01.

WRITE 'Hello ABAP!'.

WRITE / '이 줄은 새 줄에 출력됩니다.'.   " /  = 줄바꿈

* 콜론(:)을 쓰면 같은 명령을 여러 번 반복하지 않아도 됩니다 (체인 문장)
WRITE: / '3일 뒤에는',
       / '주문 관리 프로그램을',
       / '완성합니다'.
```

6. 저장(Ctrl+S) → 활성화(Ctrl+F3) → 실행(F8) 📷

```
[실행 결과]
Hello ABAP!
이 줄은 새 줄에 출력됩니다.
3일 뒤에는
주문 관리 프로그램을
완성합니다
```

> 💡 **에디터 팁**: 자동완성 Ctrl+Space, 문법검사 Ctrl+F2, Pretty Printer(자동 정렬) Shift+F1. 처음부터 습관 들이세요.

## U02. 변수와 데이터 타입 (10:45~11:35)

### 🎯 목표
- DATA/CONSTANTS/TYPES로 변수·상수·타입을 선언할 수 있다
- 기본 타입을 구분하고 산술 연산을 할 수 있다

### 📖 개념

**변수 선언**: `DATA 변수명 TYPE 타입.` — ABAP은 선언하지 않은 변수를 쓸 수 없습니다.

| 타입 | 의미 | 예 |
|------|------|-----|
| `c LENGTH n` | 고정 길이 문자 | 이름, 코드 |
| `string` | 가변 길이 문자열 | 긴 텍스트 |
| `n LENGTH n` | 숫자 문자 (0패딩) | 사번 '00123' |
| `i` | 정수 | 수량 |
| `p LENGTH n DECIMALS m` | 소수점 있는 수(팩형) | 금액, 단가 |
| `d` | 날짜 (YYYYMMDD 8자리) | 주문일 |
| `t` | 시간 (HHMMSS 6자리) | 등록시각 |

- 관례: 변수 이름은 `gv_`(전역 변수), `lv_`(로컬 변수), 상수는 `gc_` 접두어. 3일간 이 관례를 따릅니다.
- **시스템 필드**: SAP이 항상 채워주는 값들. 오늘은 `sy-datum`(오늘 날짜), `sy-uname`(내 ID) 두 개만. (앞으로 `sy-subrc`, `sy-tabix`, `sy-index`가 계속 나옵니다 — 그때마다 소개)

### 💻 따라치기 — `ZEDU##_U02`

```abap
REPORT zedu##_u02.

* 1) 선언
DATA: gv_prod_name TYPE c LENGTH 20,             " 제품명
      gv_qty       TYPE i,                       " 수량
      gv_price     TYPE p LENGTH 8 DECIMALS 2,   " 단가
      gv_amount    TYPE p LENGTH 10 DECIMALS 2,  " 금액
      gv_vat       TYPE p LENGTH 10 DECIMALS 2,  " 부가세
      gv_today     TYPE d.                       " 날짜

CONSTANTS: gc_vat_rate TYPE p LENGTH 3 DECIMALS 2 VALUE '0.10'.  " 부가세율(상수)

* 2) 값 넣기와 계산
gv_prod_name = '기계식 키보드'.
gv_qty       = 3.
gv_price     = '45000.00'.        " 소수 리터럴은 따옴표로
gv_amount    = gv_qty * gv_price.
gv_vat       = gv_amount * gc_vat_rate.   " WRITE에는 수식을 못 쓰므로 변수에 먼저 계산
gv_today     = sy-datum.          " 시스템 필드: 오늘 날짜

* 3) 출력
WRITE: / '제품명:', gv_prod_name,
       / '수량  :', gv_qty,
       / '단가  :', gv_price,
       / '금액  :', gv_amount,
       / '부가세:', gv_vat,
       / '날짜  :', gv_today.

* 4) CLEAR: 변수를 초기값으로
CLEAR gv_qty.
WRITE: / 'CLEAR 후 수량:', gv_qty.   " 0
```

**TYPES** — 나만의 타입 만들기 (내일 내부 테이블에서 본격 활용):

```abap
TYPES: ty_price TYPE p LENGTH 8 DECIMALS 2.
DATA:  gv_price2 TYPE ty_price.
```

> ✨ **모던 ABAP으로는 이렇게**
> ```abap
> DATA(lv_amount) = gv_qty * gv_price.              " 인라인 선언: 타입을 우변에서 추론
> WRITE / |{ gv_prod_name } { gv_qty }개 = { lv_amount }원|.   " 문자열 템플릿 | ... { 변수 } ... |
> ```
> 인라인 선언은 "처음 값이 정해지는 그 자리"에서 변수를 만듭니다. 실무 신규 코드에서 표준처럼 쓰입니다.

## U03. 조건문 IF / CASE (13:00~13:50)

### 🎯 목표
- IF / ELSEIF / ELSE 와 CASE / WHEN으로 분기할 수 있다
- 비교·논리 연산자를 쓸 수 있다

### 📖 개념

```abap
IF 조건.            CASE 변수.
  ...                 WHEN 값1.  ...
ELSEIF 조건.          WHEN 값2.  ...
  ...                 WHEN OTHERS. ...
ELSE.               ENDCASE.
  ...
ENDIF.
```

- 비교: `=  <>  >  <  >=  <=` (또는 EQ NE GT LT GE LE — 옛 코드에서 자주 만남)
- 논리: `AND` `OR` `NOT`
- **하나의 값으로 여러 갈래** → CASE, **조건식이 복잡** → IF

우리 주문에는 상태가 있습니다: **N(신규) / C(완료) / X(취소)**. 오늘 오후 SE11에서 이 값들을 도메인에 새겨 넣고, 3일 내내 만나게 됩니다.

### 💻 따라치기 — `ZEDU##_U03`

```abap
REPORT zedu##_u03.

DATA: gv_status TYPE c LENGTH 1,
      gv_text   TYPE c LENGTH 10,
      gv_qty    TYPE i,
      gv_stock  TYPE i.

* 1) CASE: 상태 코드 -> 상태명
gv_status = 'N'.

CASE gv_status.
  WHEN 'N'.
    gv_text = '신규'.
  WHEN 'C'.
    gv_text = '완료'.
  WHEN 'X'.
    gv_text = '취소'.
  WHEN OTHERS.
    gv_text = '알수없음'.
ENDCASE.

WRITE: / '상태:', gv_status, '->', gv_text.

* 2) IF: 재고 확인
gv_qty   = 5.
gv_stock = 3.

IF gv_qty <= 0.
  WRITE / '오류: 수량은 1 이상이어야 합니다.'.
ELSEIF gv_qty > gv_stock.
  WRITE: / '재고 부족! 주문', gv_qty, '/ 재고', gv_stock.
ELSE.
  WRITE / '주문 가능합니다.'.
ENDIF.

* 3) AND / OR
IF gv_qty > 0 AND gv_qty <= gv_stock.
  WRITE / '(재확인) 주문 가능'.
ENDIF.
```

> `gv_status = 'n'`으로 바꿔 실행해보세요. → '알수없음'. **따옴표 안은 대소문자를 구분**하기 때문입니다.

> ✨ **모던 ABAP으로는 이렇게**
> ```abap
> gv_text = COND #( WHEN gv_status = 'N' THEN '신규'
>                   WHEN gv_status = 'C' THEN '완료'
>                   WHEN gv_status = 'X' THEN '취소'
>                   ELSE '알수없음' ).
> ```
> COND는 "값을 돌려주는 IF". 대입 한 줄로 끝납니다.

## U04. 반복문 DO / WHILE (14:00~14:50)

### 🎯 목표
- DO / WHILE로 반복하고 sy-index를 읽을 수 있다
- EXIT / CONTINUE / CHECK로 반복을 제어할 수 있다

### 📖 개념

```abap
DO 5 TIMES.        DO.                  WHILE 조건.
  ...                IF 조건. EXIT. ENDIF.   ...
ENDDO.             ENDDO.               ENDWHILE.
```

- `sy-index`: 지금 몇 번째 반복인지 (1부터)
- `EXIT` = 반복 즉시 탈출, `CONTINUE` = 이번 회차 건너뜀, `CHECK 조건` = 조건이 거짓이면 건너뜀(CONTINUE와 동일 효과)
- ⚠️ **무한 루프 주의**: `DO.`(횟수 없음)나 WHILE에서 조건 변수를 갱신하지 않으면 프로그램이 멈추지 않습니다. 화면이 멎으면 강사를 부르세요 (메뉴 → 트랜잭션 중단)

### 💻 따라치기 — `ZEDU##_U04`

```abap
REPORT zedu##_u04.

DATA: gv_sum TYPE i,
      gv_cnt TYPE i.

* 1) DO n TIMES + sy-index
DO 5 TIMES.
  WRITE: / sy-index, '번째 반복입니다'.
ENDDO.

* 2) 1~10 누적합
CLEAR gv_sum.
DO 10 TIMES.
  gv_sum = gv_sum + sy-index.
ENDDO.
WRITE: / '1~10 합계:', gv_sum.        " 55

* 3) WHILE
CLEAR gv_cnt.
WHILE gv_cnt < 3.
  gv_cnt = gv_cnt + 1.               " 이 줄을 지우면? -> 무한 루프!
  WRITE: / 'WHILE 반복:', gv_cnt.
ENDWHILE.

* 4) CONTINUE / EXIT
DO 10 TIMES.
  IF sy-index MOD 2 = 1.
    CONTINUE.                        " 홀수는 건너뛰기
  ENDIF.
  IF sy-index > 8.
    EXIT.                            " 8 초과면 반복 종료
  ENDIF.
  WRITE: / '짝수:', sy-index.        " 2 4 6 8
ENDDO.
```

### ✍️ 퀴즈 Q05 — 3의 배수 합

## U06. SE11 ① 도메인과 데이터 엘리먼트 (15:55~16:45)

### 🎯 목표
- 도메인 → 데이터 엘리먼트 → 필드 3계층을 이해한다
- 고정값을 가진 도메인과 데이터 엘리먼트를 만들 수 있다

### 📖 개념

**데이터 딕셔너리(DDIC)** = 시스템 전체가 공유하는 "데이터 정의 창고". SE11에서 관리합니다.

```
[도메인]              [데이터 엘리먼트(DE)]        [테이블 필드]
기술 속성              의미(라벨, 문서)             실제 사용처
CHAR 1, 고정값 N/C/X → "주문상태" 라는 이름표    →  ZEDU##_ORDER-STATUS
```

- **도메인**: 타입/길이/허용값 등 **기술적** 정의. 여러 DE가 재사용
- **데이터 엘리먼트**: 도메인에 **의미**(필드 라벨, F1 도움말)를 입힌 것. 여러 테이블 필드가 재사용
- 왜 나눌까? "주문상태"와 "배송상태"는 의미(DE)는 달라도 기술 속성(CHAR1 도메인)은 같을 수 있음 → 재사용·일관성
- **데이터 엘리먼트의 라벨은 화면(ALV 컬럼 제목, 팝업 라벨)에 자동으로 나타납니다.** 내일 눈으로 확인하게 됩니다

### 💻 따라치기 — 주문번호/주문상태 도메인·DE 만들기

**(1) 도메인 ZEDU##_ORDID (주문번호)**

1. `SE11` → Domain 란에 `ZEDU##_ORDID` → [Create]
2. Short Description: `주문번호`
3. Definition 탭: Data Type `NUMC`, No. Characters `10`
4. 저장([Local Object]) → **활성화(Ctrl+F3)** 📷 — DDIC 객체도 활성화해야 쓸 수 있습니다

**(2) 도메인 ZEDU##_STATUS (주문상태 + 고정값)**

1. `ZEDU##_STATUS` → [Create], 설명 `주문상태`, Data Type `CHAR`, Length `1`
2. **Value Range 탭** → Single Values에 입력: 📷

| Fixed Value | Description |
|-------------|-------------|
| N | 신규 |
| C | 완료 |
| X | 취소 |

3. 저장 → 활성화. **이제 이 도메인을 쓰는 필드에는 N/C/X 외 값이 "정의상" 없다는 뜻**이 되고, 화면에서 F4 목록도 공짜로 생깁니다

> 고정값은 화면 입력 검사·F4의 기준이며, 프로그램의 `INSERT`/`UPDATE`까지 자동으로 막지는 않습니다. 그래서 Day 3에서 검증 FM을 직접 만듭니다.

**(3) 데이터 엘리먼트 2개**

1. SE11 → **Data type** 란에 `ZEDU##_ORDID` → [Create] → **Data Element** 선택
2. Data Type 탭: Domain 란에 `ZEDU##_ORDID` (같은 이름의 도메인 참조 — SAP에서 흔한 관례)
3. **Field Label 탭**: Short/Medium/Long/Heading 전부 `주문번호` 📷
4. 저장 → 활성화
5. 같은 방법으로 DE `ZEDU##_STATUS` (도메인 ZEDU##_STATUS, 라벨 `주문상태`) 생성

## U07. SE11 ② 스트럭처와 테이블 (16:55~17:45)

### 🎯 목표
- 투명 테이블을 생성하고 활성화할 수 있다
- 스트럭처와 테이블의 차이를 안다

### 📖 개념

- **투명 테이블(Transparent Table)**: DDIC 정의가 DB에 실제 테이블로 만들어진 것. **데이터가 저장됨**
- **스트럭처(Structure)**: 필드 묶음의 정의일 뿐, 데이터 저장 안 됨. 프로그램에서 변수/화면 구조로 사용
- **MANDT(Client)**: SAP 한 시스템 안의 "독립된 회사 공간" 구분자. 업무 테이블 1번 키는 관례적으로 MANDT
- 필드 타입 지정 3가지 방법을 오늘 다 써봅니다: ① 내가 만든 DE ② 남(공용/표준)이 만든 DE ③ DE 없이 빌트인 타입 직접
