# SAP ABAP 3일 기초교육 산출물 제3자 검토보고서

> 검토 기준: S/4HANA 2023 온프레미스의 classic ABAP(ABAP Platform 2023 계열)을 기준으로 정적 검토했다. 실제 대상 시스템에서의 활성화·실행 테스트는 수행하지 않았다.

## 1. 총평

전체 설계 흐름과 3일간의 누적 학습 구조는 탄탄하며, classic 문법도 S/4HANA 2023에서 대체로 그대로 사용할 수 있다.
다만 완성 코드가 취소(X) 주문을 N/C로 되돌릴 수 있어 확정 업무규칙을 위반하므로 현 상태 그대로 강의하는 것은 권하지 않는다.
모던 ABAP 박스 2곳은 그대로 복사하면 활성화되지 않거나 완결되지 않으며, 일부 설명은 실제 ABAP 동작보다 과도하게 단순화되어 있다.
표준 FM 파라미터와 `sval`/`slis_*`/`sytabix` 사용 형태는 정적 대조상 타당하지만, PF-STATUS 복사 경로는 대상 2023 시스템에서 사전 실동작 확인이 필요하다.

## 5. 총괄 판정

**판정: 최소 수정 후 강의 가능. 현재 그대로는 비권장.** S/4HANA 2023 기준으로 classic Open SQL, `TYPE-POOLS: slis`, FORM/PERFORM, `REUSE_ALV_GRID_DISPLAY`, `POPUP_GET_VALUES`, `POPUP_TO_CONFIRM` 사용 자체는 버전상 큰 문제가 없고, 7.40+ 보조 구문도 지원 범위 안이다. 다만 아래 최소 조치는 강의 전에 완료해야 한다.

1. TP-3 스켈레톤과 Day 3 부록 B에서 X 주문 수정 차단.
2. U15 SALV 박스에 `CX_SALV_MSG` 처리 추가, U09 VALUE 예제 타입 선언 보완.
3. 대상 S/4HANA 2023 시스템에서 부록 B 활성화·CRUD 통합 테스트 및 `SAPLKKBL/STANDARD_FULLSCREEN` 복사 스모크 테스트.
4. 주문 샘플 20건의 실제 값을 확정·수록하고 기본 90일 조건에서 기대 건수 검증.
5. Q10~Q12 스켈레톤의 누락 선언을 보완하고 U16/U18 프로그램 목록을 정합화.

권장 통합 시험은 “20건 로드 → 기본 조회 → X 행 수정 거부 → 정상 등록 → N 주문 수정/완료 전환 → C 재수정 거부 → N 삭제 → 리셋 후 20건 복구” 순서다.
