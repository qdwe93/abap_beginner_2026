# SAP ABAP 3일 기초교육 — 커리큘럼 (STEP 1 초안)

> 기준 문서: [00_요청프롬프트.md](00_요청프롬프트.md)
> 시간: 매일 09:00~18:00, 점심 12:00~13:00, 실질 8시간
> 수강생: 경험자·초보 혼재 → 기본 진도는 초보 기준, 심화 퀴즈로 경험자 대응

---

## 설계 원칙

1. **유닛 구조**: 모든 유닛(U01~U19)은 `개념(20~30분) → 따라치기 실습 → 퀴즈` 사이클. 표의 시간은 3단계 합산
2. **스켈레톤 방식**: 실습·퀴즈는 전후 코드가 완성된 스켈레톤에서 `" TODO` 부분만 직접 작성
3. **Toy Project 역산 설계**: Day 1의 SE11 실습에서 만드는 테이블 = toy project 테이블.
   Day 2 마지막 종합실습에서 만드는 조회 리포트 = toy project의 조회 화면.
   → 매일의 결과물이 Day 3 오후에 하나로 합쳐지는 구조
4. **✨ 보조 커리큘럼**: 신문법은 유닛 말미 5분 내외의 "모던 ABAP으로는 이렇게" 박스. 진도가 밀리면 생략 가능(교재에는 수록되므로 자습 가능)
5. **혼재 수준 대응**: 각 유닛 퀴즈에 (a) 기본 (b) 심화 두 단계 상시 제공. 빨리 끝낸 경험자는 심화로, 못 따라온 초보는 강사가 기본 퀴즈 해설로 수습
6. **네이밍**: 실습 프로그램 `ZEDU##_U01` 형식(## = 수강생 번호), 테이블 등은 STEP 2에서 확정

## 진도 리스크 대응표 (강사용 요약 — 상세는 STEP 5 강사 가이드)

**진도가 밀릴 때 줄이는 순서** (앞 번호부터 희생):

1. ✨ 보조 박스 전부 (교재로 자습 가능)
2. U12 Open SQL ② → JOIN 따라치기만 하고 퀴즈 생략
3. U05 디버거 → 브레이크포인트+변수확인만 15분 축약
4. TP-4 삭제 → 완성 코드 배포 후 설명만
5. 미니 종합퀴즈 ①, 복습 퀴즈 → 축소

**절대 사수하는 것**: U07(테이블 생성), U09~U11, U13, U15(ALV), U17(CRUD), TP-1~TP-3
→ 이것이 빠지면 toy project가 완성되지 않음

**진도가 빠를 때**: 각 유닛 심화 퀴즈(b형) 투입 → 그래도 남으면 TP-5 확장 과제(검색도움말, 더블클릭 이벤트, 합계 행 등)
