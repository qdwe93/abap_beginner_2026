## 1. 목적과 사용 맥락

- **용도**: 강사가 3일 강의 중 빔프로젝터로 띄워 놓고 진행하는 **강의 진행용 슬라이드**. 수강생 배포용 교재(10~12)와 실습 소스(40_패키지)는 이미 있으므로, 슬라이드는 "그 시간에 무엇을 왜 하는지"를 전달하고 라이브 시연을 안내하는 역할에 집중한다.
- **화면 운용**: 강의장 화면은 슬라이드 ↔ SAP GUI 시연을 수시로 전환한다. 슬라이드는 시연을 대체하지 않고 시연의 **before(개념·목표)와 after(요약·체크포인트)** 를 담당한다.
- **커버 범위**: 강의 중 가르치는 **모든 세그먼트**를 포함한다 — OT, U01~U19, 미니 종합퀴즈①, 복습 퀴즈 R2/R3, 종합실습②, TP-0~TP-5, Day별 체크리스트, 수료 랩업.

## 2. 설계 원칙 (제작 전 확정)

1. **교재가 원본(Single Source of Truth)**: 슬라이드의 모든 개념 설명·코드·시간·프로그램명은 교재 10~12에서 파생한다. 슬라이드에서 새 내용을 창작하지 않는다. 슬라이드마다 발표자 노트에 출처 절(예: `10_교재_Day1.md > U02 > 개념`)을 기록해 정합성 검증이 가능하게 한다.
2. **코드 전문은 싣지 않는다**: 긴 따라치기 코드 전문은 교재와 패키지 스켈레톤이 담당한다. 슬라이드에는 **핵심 조각(3~10줄)** 과 "교재 해당 절을 펴세요" 안내만 싣는다. 이중 유지보수로 인한 교재↔슬라이드 코드 불일치가 이 프로젝트 최대 품질 리스크이므로 원천 차단한다. 예외: TP-2~4의 `★ TODO` 요구사항 요약은 슬라이드에 명시(학생이 화면을 보며 작성하는 대상이므로).
3. **퀴즈 정답 비노출**: 정답은 교재 부록 A에만 있다는 기존 정보 분리 원칙을 유지한다. 퀴즈 슬라이드는 문제·힌트·제한시간만 싣고, 해설은 강사가 정답 풀이 시점에 SAP GUI 라이브 코딩으로 보여준다. (해설용 정답 슬라이드가 필요하면 **강사용 숨김 슬라이드**로 분리해 배포본에서 제외)
4. **확정 설계 보존**: 94 지시서 6장의 확정 설계를 슬라이드에도 그대로 적용한다 — classic 문법 본문 + 신문법은 보조 박스, `REUSE_ALV_GRID_DISPLAY` 유지, 취소 기능 없음, X 차단 리터럴 메시지(023 없음), MAX+1 채번, `##` 학생 번호 표기, LIST → U19 → MAIN 복사 체인 등.
5. **운영 장치 내장**: 강사가이드의 시간 안전장치·기준본 전환·E/D 수준 분기를 슬라이드에 시각 요소(타이머 배너, 전환 문구)로 넣어, 슬라이드만 따라가도 운영 규칙이 지켜지게 한다.
6. **한 슬라이드 한 메시지**: 개념 1개 또는 지시 1개. 교재 한 절을 그대로 옮겨 붙이지 않는다.

## 6. 코드·화면 표시 규칙

1. `##` 학생 번호 자리표시자를 코드·프로그램명에 그대로 유지하고, Day 1 OT에서 "##는 각자 번호로" 규칙을 한 번 각인시킨다.
2. 코드 조각은 **고정폭 폰트 + 문법 하이라이트**, 슬라이드당 최대 10줄. 그 이상이면 교재 참조로 대체.
3. 스켈레톤의 `* TODO`와 TP의 `★ TODO`는 강조색으로 표시해 "학생이 쓸 부분"임을 시각화한다.
4. 신문법(✨)은 본문과 다른 배경색 박스로 구분해 "참고용, 오늘 실습은 classic"임을 명시한다.
5. 위험 패턴 3종(WRITE 안 산술식 / CONCATENATE에 숫자형 / ORDER BY에 집계함수)은 해당 유닛(U02, U12, U19)에서 ❌→✅ 대비 슬라이드로 다룬다 — 이번 93/94 수정에서 확정된 교육 포인트.
