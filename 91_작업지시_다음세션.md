## 2. ⚠️ 의도적 설계 — 수정 금지 목록

다음은 검토보고서가 지적했거나 지적할 법하지만, **사용자와 합의된 의도적 설계**다. "개선"하지 말 것:

1. **취소(X) 처리 기능 미구현**: 기본 과정에서 X는 조회·필터 연습용으로만 존재. 취소/취소해제 기능은 확장 과제. 이번에 추가하는 것은 "X 수정 차단" **한 가지뿐**
2. **리터럴 메시지 혼용**: 등록/수정/삭제 실패, X 차단 등은 리터럴 유지. 메시지 클래스로 통일하지 말 것 — "리터럴(임시·간단) vs 클래스(정식)" 두 방식을 다 보여주는 것 자체가 교육 목적이고, 사용자가 단순함을 선택함
3. **02 설계서 §7의 단순화 6건**: 채번 MAX+1(SNRO 아님), 단가 DEC(CURR 아님), 금액 미저장, COMMIT을 메인에서, POPUP_GET_VALUES(Dynpro 아님), 잠금 없음 — 전부 의도적. "실무에서는" 보충은 STEP 5 강사 가이드 몫
4. **REUSE_ALV_GRID_DISPLAY(클래식) 채택**: SALV/OO ALV로 바꾸지 말 것. 신문법·OO는 ✨보조 박스에만
5. **구문법 우선 원칙**: 본문 코드는 classic 문법. new Open SQL·인라인 선언 등은 ✨박스 전용
6. **프로그램 복사 체인**: LIST(Day2) → U19(버튼) → MAIN(TP). 재타이핑 최소화 장치이므로 구조 변경 금지
7. **Q03·Q05 백지 코딩**: 스켈레톤 원칙의 의도적 예외(종합 점검용). 15번 항목처럼 "의도적"임을 명시만 하고 스켈레톤화하지 말 것
8. **U08의 짧은 구성**: 15분 운영 루틴 유닛. 정식 개념→따라치기→퀴즈 사이클로 확장하지 말 것 (16번의 최소 보강만)
9. **📷 스크린샷 자리표시**: 실물 캡처는 강사가 시스템에서 직접. 채우려 하지 말 것
10. **`##` 표기**: 학생 번호 자리표시자. 코드 블록 안의 ##은 의도된 것 (교재 사용법에 정의됨)
11. **.abap 파일 분리 없음**: 코드는 현재 교재 안에만 존재. 분리는 STEP 6 패키징에서
12. **검토보고서(90_검토보고서.md)는 수정하지 말 것**: 검토 세션의 기록물
