# SAP ABAP 3일 교육과정 — 배포 패키지

## 디렉터리

| 경로 | 용도 |
|------|------|
| `학생용_스켈레톤` | 수강생에게 배포할 TODO 중심 ABAP 소스 |
| `강사용_완성` | 시연·복구·정답 확인용 완성 ABAP 소스 |
| `시스템준비` | 공용 DDIC, 로더, 준비·검증 체크리스트 |

## 사용 순서

1. `시스템준비/01_공용_DDIC_생성절차.md`에 따라 공용 객체를 준비한다.
2. `ZEDU_MASTER_LOAD.abap`으로 고객 5건·제품 8건을 투입한다.
3. 학생별 U06~U07 객체를 생성하거나 강의 전 복구 기준 객체를 준비한다.
4. `ZEDU_DATA_LOAD.abap`으로 학생별 주문 20건을 투입한다.
5. 유닛별 스켈레톤을 배포하고 필요할 때 해당 완성 소스를 기준본으로 사용한다.
6. 강의 전에 `04_강의전_준비체크리스트.md`의 시스템 검증을 완료한다.

## 파일명 규칙

- 파일명과 코드의 `##`는 학생 번호 두 자리 자리표시자다.
- 실제 SAP 객체 생성 시 학생 번호로 바꾼다.
- `ZEDU_MAIN_DEMO`는 강사 슬롯 `00`의 객체(`ZEDU00_*`)를 사용한다.

## TP 스켈레톤(`TP_ZEDU##_MAIN.abap`) 사용법

- 이 파일은 새 프로그램 생성용 독립 골격이 아니다. 확정 복사 체인 `ZEDU##_LIST → ZEDU##_U19 → ZEDU##_MAIN`에 따라, SE38에서 `ZEDU##_U19`를 `ZEDU##_MAIN`으로 GUI Status·Text Elements와 함께 복사한 뒤 사용하는 복구/대조용 소스다.
- PF-STATUS `MAIN`은 `.abap` 텍스트만으로 생성되지 않으므로 SE38 복사 옵션의 GUI Status 체크가 필수다.
- 소스는 U19 복사 직후 상태(선택화면·조회·ALV·콜백·PF-STATUS 참조 완성)이며, TP-2~TP-4의 create/change/delete FORM만 단계별 TODO로 남아 있다.

## 소스가 없는 유닛

| 유닛 | 이유 | 패키지 자료 |
|------|------|-------------|
| U06 | 도메인·데이터 엘리먼트 생성 | `시스템준비/02_학생별_DDIC_대조표.md` |
| U07 | 테이블·스트럭처 생성 | `시스템준비/02_학생별_DDIC_대조표.md` |
| U08 | SE16N·로더 실행 운영 루틴 | `시스템준비/ZEDU_DATA_LOAD.abap` |

U05는 U03~U04 코드를 디버깅하는 유닛이지만 수업 편의를 위해 별도 버그 스켈레톤과 수정본을 포함한다. 교재 Q06의 `ZEDU##_U05Q`에 해당하며, 배포 시 파일의 프로그램명(`ZEDU##_U05_BUG`/`_FIXED`)을 그대로 사용해도 된다.

U17 소스 쌍은 교재의 라운드 1~3 전체(INSERT/UPDATE/UPDATE SET/MODIFY/ROLLBACK 체험/DELETE)를 포함한다.

## 정합성 기준

- 학생별 주문 20건은 `10_교재_Day1.md` 부록 B와 `ZEDU_DATA_LOAD.abap`이 동일해야 한다.
- 상태 분포는 N 10건/C 6건/X 4건이다.
- X 주문 수정 차단은 리터럴 메시지를 사용하며 메시지 023을 만들지 않는다.
- 기본 ALV는 `REUSE_ALV_GRID_DISPLAY`를 유지한다.
- 실제 시스템 활성화·실행 검증 결과는 준비 체크리스트에 기록한다.
