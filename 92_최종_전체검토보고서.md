## 3. 심각도별 이슈

### 🔴 블로커 (1건)

**[🔴] `10_교재_Day1.md` 169행, U02 따라치기 `ZEDU##_U02` | WRITE 목록에 산술 표현식 직접 사용 | 근거: `WRITE: ... / '부가세:', gv_amount * gc_vat_rate,` — classic ABAP의 `WRITE` 피연산자 위치는 데이터 객체(변수·리터럴)만 허용하고 산술 표현식 위치가 아니므로 문법 오류(활성화 실패)가 강하게 예상된다. Day 1 오전 전원 따라치기 코드라 실패 시 수업이 즉시 막힌다. 참고로 패키지의 강사용 완성본 `U02_ZEDU##_U02.abap`에는 부가세 줄이 없어 교재만 해당된다 | 수정안: 변수를 도입한다 —**

```abap
DATA gv_vat TYPE p LENGTH 10 DECIMALS 2.
...
gv_vat = gv_amount * gc_vat_rate.
WRITE: ...
       / '부가세:', gv_vat,
```

(또는 부가세 출력 줄 삭제. 미니 종합퀴즈 ①의 "부가세 10% 줄 추가" 확장 안내도 "변수에 계산 후 출력" 힌트를 덧붙이면 같은 함정을 피한다.)
※ 시스템 미실행 상태의 정적 판단이므로, 수정 전 실제 시스템에서 원문 활성화를 시도해 확정해도 된다. 다만 수정안 쪽이 어느 시스템에서도 안전하다.

### 🟡 중요 (5건)

**[🟡] `20_퀴즈집.md` 650~656행, D12 정답 | classic 구문의 ORDER BY에 집계함수 직접 사용 | 근거: `ORDER BY COUNT( * ) DESCENDING` — 콤마 없는 구문법 Open SQL의 ORDER BY는 컬럼명 또는 SELECT 목록의 별칭만 허용하며 집계식 직접 지정은 문법 오류 위험이 높다 | 수정안(택1): ① 별칭 사용 — `SELECT a~cust_id b~cust_name COUNT( * ) AS order_cnt ... GROUP BY a~cust_id b~cust_name ORDER BY order_cnt DESCENDING.` ② 전체 GROUP BY 조회 후 `SORT gt_rank BY order_cnt DESCENDING.` + 상위 3건만 출력(이 경우 `UP TO 3 ROWS` 제거). 해설의 "동률 보조 정렬" 문구는 유지 가능**

**[🟡] `20_퀴즈집.md` 1081~1089행, D19 정답 | CONCATENATE에 P(DEC) 타입 인자 | 근거: `CONCATENATE ... gs_list-amount INTO lv_msg ...` — `AMOUNT`는 `ZEDU_AMOUNT`(DEC 15,2 → ABAP P 타입)이고 CONCATENATE 인자는 문자형(c/n/d/t/string)만 허용되므로 문법 오류 | 수정안: 문자 변수로 변환 후 연결 —**

```abap
DATA lv_amt TYPE c LENGTH 20.
lv_amt = gs_list-amount.        " 또는 WRITE gs_list-amount TO lv_amt.
CONDENSE lv_amt.
CONCATENATE '주문' gs_list-order_id '금액' lv_amt
  INTO lv_msg SEPARATED BY space.
```

**[🟡] `20_퀴즈집.md` 1182~1187행, 확장 과제 2(&IC1) 풀이 예시 | CONCATENATE에 INT4 타입 인자 | 근거: `'수량:' gs_list-quantity` — `QUANTITY`는 `ZEDU_QTY`(INT4 → i)로 문자형이 아니어서 문법 오류. 같은 문장의 order_id(NUMC)·cust_name·prod_name·status는 문자형이라 문제없음 | 수정안: D19와 동일 패턴 — `DATA lv_qty_c TYPE c LENGTH 10. lv_qty_c = gs_list-quantity. ## 5. ABAP 핵심 소스별 검토 결과 (정적)

### 5-1. `시스템준비/ZEDU_DATA_LOAD.abap` — 집중 검토, 이슈 없음

- 동적 처리: `CONCATENATE 'ZEDU' p_no '_ORDER'`(p_no TYPE n LENGTH 2 → 0패딩 보장), `CREATE DATA gr_table TYPE STANDARD TABLE OF (gv_tabname)` + `CATCH cx_sy_create_data_error`(테이블 미존재·비활성 시 정확한 예외), `ASSIGN gr_table->*` — 표준 패턴으로 타당.
- 시드 주입: `DEFINE add_seed` 매크로, `ASSIGN COMPONENT ... OF STRUCTURE` 필드 단위 대입 — MANDT 명시 대입은 Open SQL 자동 클라이언트 처리와 충돌하지 않음(무해). `sy-datum - gs_seed-days`를 DATS 필드 심볼에 대입하는 날짜 연산은 교재 U14의 `sy-datum - 90`과 같은 표준 변환 경로.
- 초기화/중복키: `DELETE FROM (gv_tabname)`(WHERE 없는 전체 삭제 — 동적 테이블명 구문 유효), `INSERT ... FROM TABLE <gt_order> ACCEPTING DUPLICATE KEYS` — 리셋 미체크+기존 데이터 시 덤프 없이 `sy-subrc=4`로 떨어지고, `sy-subrc = 0 AND sy-dbcnt = 20` 조건 미달이면 ROLLBACK(부분 삽입까지 원복) + 원인 안내 E 메시지. **중복키 처리 설계가 안전하다.**
- COMMIT/ROLLBACK 위치·메시지 타입(E로 중단) 적절.
