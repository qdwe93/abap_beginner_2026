# SAP ABAP 3일 기초교육 과정 개발 요청 프롬프트

> 이 문서는 교육과정 개발 전체를 관통하는 기준 문서입니다.
> 이후 만들어지는 모든 산출물(커리큘럼·교재·퀴즈·강사자료)은 이 문서의 요구사항을 따릅니다.

---

## 1. 배경 및 교육 환경

- **강사**: SAP ABAP 기초교육 담당, 3일간 오프라인 실습 교육
- **시스템 환경**: **S/4HANA 2023 (온프레미스)**, SAP GUI 기반 — 확정
- **언어 범위**: Classic ABAP (리포트 프로그램 중심)
  - Cloud ABAP / BTP / RAP / CDS 개발은 범위 제외
- **문법 정책**: 구문법(classic syntax)을 주 커리큘럼으로 먼저 가르치고,
  신문법(7.40+ new syntax, new Open SQL)은 **보조 커리큘럼**으로 배치
  - 형태 제안: 각 단원 말미에 "✨ 모던 ABAP으로는 이렇게" 박스로 구문 대비표 + 짧은 실습
  - 예: `DATA lv_x TYPE i. lv_x = 1.` → `DATA(lv_x) = 1`, `SELECT ... INTO TABLE @DATA(lt_...)`, `VALUE #( )`, 문자열 템플릿 `|...|`, `READ TABLE` → `lt_itab[ ... ]` 등

## 2. 교육 대상 ✅ 확정

- 수강생 수준: **경험자·초보 혼재** → 기본 진도는 초보 기준, 심화 퀴즈로 경험자 대응
- 1일 교육 시간: **09:00~18:00 (점심 1시간, 실질 8시간)**
- 인원 수: 미정 (확정되면 반영)
- 실습 계정: 수강생별 SAP 계정 + 개발자 키 발급 전제, 네이밍 규칙(예: `ZEDU_##_*`)은 준비 체크리스트에서 확정

## 3. 교수법 원칙 (필수 준수)

1. **짧은 사이클 반복**: 개념 설명 20~30분 → 따라치기 실습 → 퀴즈 풀이. 이 3단계가 하나의 학습 유닛
2. **개념 집중형 실습**: 전후 코드는 완성된 스켈레톤으로 제공하고, 해당 유닛에서 배우는 부분만 직접 타이핑하도록 설계
   - 스켈레톤에는 `" TODO: 여기에 LOOP 문을 작성하세요` 식의 마커 사용
3. **따라치기 → 응용 퀴즈** 순서: 강사 시연 코드를 그대로 따라 친 후, 같은 개념을 변형한 퀴즈를 스스로 해결
4. 각 유닛은 독립적으로 완결되되, Day 3 toy project에서 전부 연결되는 구조

## 4. 학습 목표 (수료 시점에 할 수 있어야 하는 것)

1. **SE11**: 도메인, 데이터 엘리먼트, 스트럭처, 테이블(+ 필드/키/기술설정)을 직접 생성
2. **SE38**: 리포트 프로그램 생성, 리포트 이벤트 흐름(INITIALIZATION → AT SELECTION-SCREEN → START-OF-SELECTION 등) 이해
3. **Open SQL**: SELECT (단건/복수건, WHERE, INTO TABLE, JOIN 기초) 활용
4. **선택화면**: PARAMETERS, SELECT-OPTIONS로 조회 조건 구성
5. **내부 테이블**: 선언, APPEND/READ/LOOP/MODIFY/DELETE/SORT 등 조작
6. **SE37**: Function Module 생성 및 호출 (IMPORTING/EXPORTING/EXCEPTIONS 실습, TABLES·CHANGING은 소개)
7. **DB CRUD**: INSERT / UPDATE / MODIFY / DELETE + COMMIT WORK 개념
8. **화면 이벤트**: PF-STATUS로 버튼 추가, AT USER-COMMAND 이벤트 처리 (**ALV 콜백 방식으로 실습**)
9. **최종 Toy Project**: 위 전부를 활용한 "실제 쓸 법한" 간단한 프로그램 완성
   - 후보 예시: 도서 대여 관리 / 사내 비품 신청 관리 / 간이 주문 관리 (선택 확인 필요)
   - 구성: 조회 화면(SELECT-OPTIONS) → 목록 출력 → 버튼으로 등록/수정/삭제

## 6. 강사용 가이드에 포함할 내용

- 유닛별 진행 팁 (시간 배분, 강조 포인트, 학생들이 자주 막히는 지점과 대처법)
- 예상 질문 & 모범 답변
- SAP 생태계/모듈 소개 (FI/CO/SD/MM/PP 등 — ABAP 개발자가 알아야 할 수준으로)
- 주요 T-code 모음 (SE11, SE38, SE37, SE80, SE16N, ST22, SM30, SM37, SU53 등 + 용도)
- 개발/운영 실무 상식: 트랜스포트(CTS) 개념, DEV→QAS→PRD 랜드스케이프, 패키지($TMP vs Z패키지), 네이밍 컨벤션, 개발자 키
- 조심해야 할 것들: `SELECT *` 남용, LOOP 안의 SELECT, COMMIT WORK 위치, 표준 테이블 직접 UPDATE 금지, 운영 장애 사례 등
- 여담/동기부여 소재: ABAP 개발자 커리어, 디버깅 일화 등 쉬는 시간용 이야깃거리
