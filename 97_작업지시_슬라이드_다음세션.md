## 2. 반드시 지킬 제작 원칙 (기획안 2장 요약 + Day 1에서 확립된 규칙)

1. **교재가 원본.** 모든 개념·코드·시간·프로그램명은 `11/12_교재`에서 파생한다. 슬라이드에서 코드를 창작하지 않는다. 각 슬라이드 발표자 노트에 강사가이드의 시간배분·강조·자주막힘·예상질문을 옮긴다.
2. **코드 전문 금지.** 긴 따라치기 코드는 교재·패키지가 담당. 슬라이드에는 핵심 조각 3~12줄만. 예외: TP-2~4의 `★ TODO` 요구사항 요약(학생이 화면 보며 작성).
3. **퀴즈 정답 비노출.** 문제·힌트·기대결과·제한시간만. 정답은 교재 부록 A에만.
4. **확정 설계 보존** (94 지시서 6장): classic 본문 + ✨신문법은 보조 박스, `REUSE_ALV_GRID_DISPLAY` 유지, 취소 기능 없음(TP-5 확장), X 수정 차단은 리터럴 메시지(메시지 023 없음), MAX+1 채번, DEC 단가·금액 미저장·메인 COMMIT, POPUP_GET_VALUES, 잠금 없음, LIST → U19 → MAIN 복사 체인, `##` 학생 번호 표기, U17 5단계 TODO, U16 DEFAULT 3종.
5. **운영 장치 내장**: 시간 안전장치는 타이머 배너, 기준본 전환 문구는 (숨김 또는 일반) 슬라이드로. Day 2·3의 안전장치 위치는 강사가이드 2-1 표 참고:
   - Day 3 U16 시작 후 30분/48분, U19 시작 후 25분, TP-2 시작 후 15/25/35분. (Day 2엔 강제 체크포인트가 없고 종합실습②가 "절대 줄이지 않을 구간".)
6. **위험 패턴 ❌→✅ 슬라이드**: 이번 과정에서 확정된 교육 포인트를 해당 유닛에서 대비 슬라이드로 명시.
   - **Day 2 U12**: classic ORDER BY에 집계함수 직접 사용 금지 → `COUNT( * ) AS order_cnt` 별칭 후 별칭으로 정렬(D12 패턴).
   - **Day 2 D10(퀴즈집)**: `WRITE`에 `gv_before - gv_after` 수식 금지 → `gv_deleted`에 계산 후 출력.
   - **Day 2 U12/U15 등 CONCATENATE**: P·INT4 타입은 문자 변환(CONDENSE) 후 연결(D19·확장2 패턴). NUMC인 order_id, `TYPE n`은 문자형이라 직접 사용 가능.

## 3. Day 1 생성 스크립트 재사용 방법

Day 1은 pptxgenjs(Node) 스크립트로 생성했다. **동일한 헬퍼/팔레트/마스터를 재사용**하는 것이 일관성의 핵심이다.

- **스크립트가 저장소에 보존되어 있다: `50_슬라이드/_src/day1_slides.js`.** 이 파일을 복제해 Day 2·3용으로 유닛 데이터만 바꿔 쓰는 것을 강력 권장한다(헬퍼·팔레트·마스터가 이미 검증됨). 출력 경로 상수(`OUT`)만 Day 2·3용으로 바꾸면 된다.
- 스크립트가 없거나 재작성이 필요하면 아래 스펙을 따른다.

### 핵심 스펙 (재작성 시)
- 라이브러리: `pptxgenjs` (npm 전역 설치). `pres.layout = "LAYOUT_WIDE"` (13.33×7.5).
- 색: NAVY `1E2761`(주) / DEEP `161D4A`(다크 배경) / ICE `CADCFC` / TINT `EEF3FB`(라이트 카드) / INK `1B2340`(본문) / MUTED `5B6478` / ACCENT `E8A33D`(TODO·타이머·강조) / RED `C4443C` / GREEN `3E7C4F` / CODEBG `151B33` / CODEFG `E8EEFF`. 코드 폰트 Courier New, 본문 Malgun Gothic.
- 색상 hex에 `#` 금지, 8자리 금지(파일 손상). 그림자 offset ≥ 0.
- 슬라이드 마스터 2종: `LIGHT`(흰 배경, 하단 좌 푸터 + 우 페이지번호), `DARK`(딥네이비 배경).
- 재사용 헬퍼: `header`(유닛칩+제목+종류태그), `unitTitle`(다크 유닛 표지), `codeBlock`(다크 코드카드, 주석 흐림처리), `shot`(📷 회색 점선 자리표시자), `bullets`, `tbl`(헤더 네이비), `banner`(앰버 칩; **전면 스트라이프 금지 — 반드시 rounded chip**), `card`.
- **디자인 금기**(pptx 스킬): 제목 밑줄·전면 색상바·사이드 스트라이프 금지. 텍스트 오버플로우 금지. 코드블록 12줄 이하, 넘으면 fontSize 10.5로.

### 유닛당 표준 패턴 (기획안 4장)
유닛 타이틀(다크) → 개념 1~3 → 라이브 시연 안내 → 흔한 에러 → (해당 유닛만)✨ → 퀴즈(정답 없음) → 체크포인트/브릿지. 유닛당 6~9장.

### 생성·검증·QA 절차 (Day 1에서 사용한 그대로)
1. Node 스크립트 실행 시 `NODE_PATH`를 npm 전역 root로 지정.
2. 구조 검증: pptx 스킬의 `office/validate.py` 실행. **한글 때문에 `PYTHONUTF8=1` 환경변수 필수**(안 그러면 cp949 디코드 오류로 오탐).
   - 스킬 스크립트가 경로 문제로 직접 실행이 안 되면, 스킬의 `scripts` 폴더를 scratchpad로 복사해 실행. 의존성: `pip install defusedxml lxml "markitdown[pptx]" Pillow`.
3. 렌더 QA: 이 환경엔 LibreOffice/pdftoppm이 없다. **PowerPoint COM으로 PNG export**가 동작함:
   ```
   $pp = New-Object -ComObject PowerPoint.Application
   $pres = $pp.Presentations.Open($pptx, $true, $false, $false)
   $pres.Export($outDir, "PNG", 1600, 900)
   $pres.Close(); $pp.Quit()
   ```
   export된 PNG를 Pillow로 3×4 컨택트시트로 묶어 Read 도구로 육안 검토(파일명은 `슬라이드N.PNG`).
4. 발견 결함(오버플로우·단어 중간 줄바꿈·겹침) 수정 후 재빌드·재검증.
